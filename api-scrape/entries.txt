<div class="usa-accordion-content" style="display: none" id="b-a17" tabindex="-1" aria-hidden="false"><h1 id="trek">Vesta/Moon/Mars Trek WMTS</h1><p> Here we have the collection of APIs that power the awesome <a href="https://trek.nasa.gov/tiles/apidoc/trekAPI.html?body=mars">Mars Trek</a> and <a href="https://trek.nasa.gov/tiles/apidoc/trekAPI.html?body=vesta">Vesta Trek</a> NASA web-based portals for exploration and have newly added <a href="https://trek.nasa.gov/tiles/apidoc/trekAPI.html?body=moon">Moon Trek</a>. These APIs can be leveraged using your favorite OGC RESTFul Web Map and Tile Service (WMTS) client. Please visit <a href="http://www.opengeospatial.org/standards/wmts">http://www.opengeospatial.org/standards/wmts</a> for more information about WMTS. To help you get started, we've included some demos using the ESRI javascript client library for <a href="https://api.nasa.gov/mars-wmts/catalog/">Mars</a> and <a href="https://api.nasa.gov/vesta-wmts/catalog/">Vesta</a>. </p><p> This API is maintained and provided by the NASA <a href="http://sservi.nasa.gov/">Solar System Exploration Research Virtual Institute (SSERVI)</a> and the <a href="mailto:MarsTrek@jpl.nasa.gov">Jet Propulsion Lab (JPL) Trek team</a>. </p><h3 id="query-parameters">Available Moon Mosaics</h3><p> Here is basic information about how to request image tiles from a RESTful WMTS service. The basic URL template is as follows: </p><p><code>http://{WMTS endpoint}/1.0.0/{Style}/{TileMatrixSet}/{TileMatrix}/{TileRow}/{TileCol}.png</code></p><p> * Some services are .png and some are .jpg. Please look at WMTS GetCapabilities ResourcesURL element to find out what extension to use. In order to fill in the variables in the template URL, you need to parse the WMTS GetCapabilities XML. GetCapabilities XML can be found for each product below. From the WMTS GetCapabilities XML, find ows:Identifier element inside Style element. This value replaces{Style}. The ows:Identifier element inside TileMatrixSet element replaces{TileMatrixSet}. Inside the TileMatrixSet element in WMTS Capabilities, there is a list of TileMatrix. This is the Zoom level. Replace{TileMatrix}with the ows:Identifier found in TileMatrix element.{TileRow}and{TileCol}are row and col index for tiles. The first zoom level for a product that covers the entire globe will have two columns and one row so that the URLs are as follows: </p><p><code>http://moontrek.jpl.nasa.gov/trektiles/Moon/EQ/LRO_WAC_Mosaic_Global_303ppd_v02/1.0.0/default/default028mm/0/0/0.jpg <br> http://moontrek.jpl.nasa.gov/trektiles/Moon/EQ/LRO_WAC_Mosaic_Global_303ppd_v02/1.0.0/default/default028mm/0/0/1.jpg</code></p><p> The second zoom level has four columns and two rows and so on. If the image tile is missing, that means there is no data coverage in that area. You can also use coverage Bbox to calculate the coverage before requesting image tiles. </p><p> For a complete listing of the Moon's Equirectangular, North Polar and South Polar based Services, please see <a href="https://trek.nasa.gov/tiles/apidoc/trekAPI.html?body=moon">https://trek.nasa.gov/tiles/apidoc/trekAPI.html?body=moon</a></p><h3 id="query-parameters">Available Mars
